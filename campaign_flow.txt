AI Campaign Builder Flow & Architecture
========================================
This document outlines the complete 10-step AI Campaign Creation workflow implemented in the StyleHive CRM, mapped to the underlying backend API calls.

Step 1: Goal Definition
-----------------------
- UI: User describes their marketing goal (e.g., "Bring back churned users").
- Backend: Handled entirely in frontend state (Zustand `useCreateCampaignStore`).

Step 2: AI Strategy Planner
---------------------------
- UI: AI recommends a strategy based on the goal.
- API Call: `POST /api/ai/strategy`
  - Body: `{ goal: "..." }`
  - Logic: Calls Gemini AI to generate `target_audience`, `recommended_offer`, and `recommended_channels`.

Step 3: AI Audience Builder
---------------------------
- UI: AI generates specific audience SQL criteria.
- API Call: `POST /api/audience/find`
  - Logic: Gemini translates the natural language audience description into safe SQL.
  - Returns: `{ customer_ids[], count, potential_revenue }` by querying the Supabase `customers` and `orders` tables.

Step 4: AI Audience Insights
----------------------------
- UI: Data breakdown of the matched audience (demographics, cities, interests).
- API Call: `GET /api/audience/insights?campaign_id=XXX`
  - Logic: Aggregates data from `customers`, `order_items`, `products`, and `categories` for the matched audience.

Step 5: AI Channel Creator
--------------------------
- UI: AI generates content for selected channels (WhatsApp, Instagram, etc.).
- API Call: `POST /api/ai/content`
  - Body: `{ channel, audience_profile, offer }`
  - Returns: Copywriting tailored to the channel, along with expected performance metrics.

Step 6: AI Creative Generator
-----------------------------
- UI: Visual assets generated by AI.
- API Call: `POST /api/ai/creatives`
  - Body: `{ campaign_name, offer, channels }`
  - Returns: Image URLs generated by AI models and hosted on Cloudinary.

Step 7: AI Personalization
--------------------------
- UI: Demonstrates 1:1 personalization for a specific customer.
- API Call: `POST /api/ai/personalize`
  - Body: `{ template, customer_profile }`
  - Returns: A dynamically personalized copy (e.g., swapping generic placeholders for past purchase references).

Step 8: Campaign Preview
------------------------
- UI: Final review of all assets and settings before launch.
- Backend: Draft is held in memory or fetched via `GET /api/campaigns/:id` if saved previously.

Step 9: Launch Campaign
-----------------------
- UI: Campaign launch sequence.
- API Call: `POST /api/campaigns`
  - Body: `{ name, goal, audience_size, potential_revenue, offer, channels, strategy, status: 'Active' }`
  - Logic: Saves the campaign to Supabase and initializes background jobs if necessary.

Step 10: Channel Simulator (The Webhook Loop)
---------------------------------------------
- UI: Real-time simulation of sending out messages.
- Architecture:
  1. Frontend hits `POST /api/campaigns/:id/simulate`.
  2. Backend pushes a job to `pg-boss` (Persistent Job Queue).
  3. The `pg-boss` worker calls the stubbed `channelController.js` to simulate a network request to WhatsApp/Instagram.
  4. The stubbed channel delays and asynchronously fires a `POST` back to `webhookController.js`.
  5. The webhook receiver updates the Supabase analytics table with "Delivered", "Opened", or "Clicked" events.
  6. Frontend polls or listens for these updates to render the live progress bars.
